#### 116. 创建和上面一样的 Pod，分别在 Pod 和容器层面设置 securityContext，容器层面的 securityContext 会覆盖掉 Pod 层面的设置

- yaml 文件

```yaml
apiVersion: v1
kind: Pod
metadata:
  creationTimestamp: null
  labels:
    run: secbusybox
  name: secbusybox
spec:
  # Pod 设置
  securityContext:
    runAsUser: 1000
  containers:
  - args:
    - /bin/sh
    - -c
    - sleep 3600;
    image: busybox
    # 容器层面的设置
    securityContext:
      runAsUser: 2000
    name: secbusybox
    resources: {}
  dnsPolicy: ClusterFirst
  restartPolicy: Never
